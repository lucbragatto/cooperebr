Gaps criticos consolidados (13 do txt Luciano)

1. Quadro 1 - Cadastro SEM_UC invisivel na UI

P1 Origem: SA1 inventario + quadro 1 do txt

O que e: enum TipoCooperado.SEM_UC existe no schema, service trata em 5 pontos, endpoint aceita. UI nunca expoe a escolha — wizard obriga upload fatura, /cadastro publico idem.

Bloqueio: caso de uso "rede de indicadores remunerada em tokens" (D-44 MST, Sprint CT). Custo fix: 4-6h Code.

2. Quadro 2 - 5 vocabularios CooperToken coexistindo

P2 D-35 + D-46.6 catalogados

O que e: mesmo conceito (escolha desconto vs tokens) tem 5 nomes: Caminho DESCONTO/CLUBE, Opcao A/B, Plano DESCONTO/Token, modoRemuneracao enum, modoToken String. Specs divergem.

Fix: Sprint CT Consolidado (21-26h) unifica vocabulario + migration 85 cooperados intermediarios.

3. Quadro 2 - MST D-44 P1 estrategica nao implementada

P1 D-44 catalogado 14/05

O que e: Mercado Secundario de Tokens - tokens proximos expiracao circulam entre cooperados via oferta onerosa. Modo E+F hibrido (deficitario prioritario + cooperativa intermedia).

Fix: Sprint dedicado 5 fases (8-13d Code). Pode virar Etapa 3 Sprint CT.

4. Quadro 3 - Cron lembrete upload documento

P1 OPERACIONAL Origem: txt quadro 3a

O que e: hoje so existe cron de auto-aprovacao apos N horas (`documentos-aprovacao.job.ts`) - faz o OPOSTO. Cooperado some sem enviar doc fica PENDENTE indefinidamente.

Fix: 3-4h Code (cron diario + template WA+email).

5. Quadro 3 - Cron aviso admin doc pendente > 3 dias

P1 Origem: txt quadro 3b

O que e: admin nao recebe resumo de quem ficou parado. Operacionalmente cego.

Fix: 2-3h Code (cron diario + agregacao + email admin).

6. Quadro 3 - Lembrete email parceiro no portal EDP

P1 Origem: txt quadro 3c

O que e: apos cadastro UC, membro precisa salvar email institucional do parceiro no portal EDP pra receber faturas. Schema tem `Cooperado.emailFaturasAtivo` mas UI nunca pede confirmacao nem cron lembra.

Fix: 2-3h Code (cron + UI confirmacao no portal cooperado).

7. Quadro 3 - Assinafy zero codigo (D-30P P1)

P1 JURIDICO

O que e: aceite hoje e flag boolean `termoAdesaoAceito` sem assinatura digital de termo + procuracao. Juridicamente fragil.

Fix: 2-3 sprints (integracao Assinafy + UI + validacoes).

8. Quadro 4 - Wizard admin D-45 P2 (4 erros encadeados)

P2 3/4 sub-fixes resolvidos commit 7bc0793

Status atual: sub-fixes 1+2+3 resolvidos 14/05. Falta sub-fix 4 (Step2Dados findByCpf antes do POST, prioridade baixa UX).

9. Quadro 5 - Realocacao inteligente multi-usina = 0%

P1 ESTRATEGICO Origem: txt quadro 5

O que e: hoje motor escolhe primeira usina com vaga (linear). Sem soma de folgas, sem analise classe GD + concessionaria, sem proposta-rascunho, sem aprovacao lista-por-lista, sem historico.

Fix: Sprint 8 dedicado (16-24h Code algoritmo + UI). Destrava onboarding 2o parceiro Sinergia.

10. Quadro 5 - Caso Exfishes D-30B P0 R$ 310k/ano ATIVO

P0 REGULATORIO

O que e: schema Usina sem campo classeGd. Migracoes nao validam cross-class (GD I -> GD II). Caso real Exfishes: prejuizo retroativo R$ 310k/ano em creditos mal compensados.

Fix: Sprint 5 schema regulatorio (4-6 dias Code). Detalhes em cadastro-usinas.html.

11. Quadros 6/7/8 - Automacao concessionaria = 0%

P1 OPERACIONAL Origem: txt quadros 6/7/8

O que e: hoje admin digita protocolo manual (`contratos.service.ts:399 ativar()`). Sem polling site EDP, sem leitor email homologacao, sem fetcher MMGD, sem trigger "fatura GD recebida -> libera cobranca". Maior gargalo operacional.

Fix: 24-32h Code (2-3 sprints dedicados).

12. Quadros 6/7/8 - Notificacao WhatsApp aprovacao concessionaria

P1

O que e: `cooperados.service.ts:836-901 aprovarConcessionaria` cria notificacao interna + envia email mas NAO dispara WhatsApp. Membro nao recebe noticia mais aguardada no canal que usa todo dia.

Fix: 1-2h Code (integrar whatsappCicloVida.notificarAprovacao).

13. Engines COMPENSADOS + DINAMICO bloqueadas

P1 BLOQUEIO Origem: SA2 inventario

O que e: engines validadas E2E sintetico 6/6 (Fase B 03/05), Decisao B33 corrigiu semantica. 7 enforcement points blindam `BLOQUEIO_MODELOS_NAO_FIXO=true` em producao. So FIXO_MENSAL roda hoje.

Pre-requisito: Sub-Fase B AMAGES (canario CREDITOS_COMPENSADOS real, 2-3 dias). Mitigacao previa D-46.SEED (`UPDATE planos SET publico=false` 3 planos legados).
