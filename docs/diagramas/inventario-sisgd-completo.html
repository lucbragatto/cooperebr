Top 20 Gaps Criticos Consolidados

1. IDOR sistemico em ~20 endpoints sem multi-tenant

P1 SEGURANCA Origem: SA7

O que e: D-48 fechou em 13/05 7 sites de Usina.findMany, mas cobrancas/contratos/faturas/motor-proposta/financeiro tem 20+ endpoints UPDATE/DELETE/relacao aninhada que nao filtram cooperativaId. Pelo menos 4 IDORs criticos confirmados por leitura.

Exemplos: GET /cobrancas/contrato/:contratoId, GET /cobrancas/:id/pdf, DELETE /cobrancas/:id, POST /contratos/:id/ativar, PATCH /faturas/documentos/:id/status, PUT /motor-proposta/tarifa-concessionaria/:id.

Impacto: Bloqueia onboarding Sinergia (2o parceiro real CoopereBR). Atacante autenticado em tenant A pode ler/alterar dados de tenant B.

2. Lembrete pra membro fazer upload de documento ate completar

P1 OPERACIONAL Origem: SA1+SA5 + txt do Luciano quadro 3

O que e: Voce descreveu no txt "nao mandou doc fica pendente e o sistema lembra ate enviar etc". Hoje existe so cron de auto-aprovacao apos N horas (documentos-aprovacao.job.ts) - faz o OPOSTO do que voce pediu.

Impacto: Cooperado aceita proposta e some sem enviar doc fica em PENDENTE_DOCUMENTOS indefinidamente. Taxa de conversao cadastro-ativacao degrada. Sem catalogacao formal. Estimativa fix: 3-4h Code

3. Lembrete pra membro confirmar email do parceiro no site da concessionaria

P1 OPERACIONAL Origem: SA1+SA5 + txt do Luciano quadro 3

O que e: Voce descreveu "salvar o email do parceiro no site da concessionaria para receber 2a via da fatura, enquanto o membro nao clicar confirmando que salvou o lembrete fica". Schema tem Cooperado.emailFaturasAtivo, email-monitor ativa flag automaticamente quando recebe fatura, mas UI nunca pede confirmacao nem cron lembra.

Impacto: Membros que nao configuram email da concessionaria ficam dependendo de OCR manual. Estimativa fix: 2-3h Code

4. Notificacao WhatsApp aprovacao concessionaria (so email hoje)

P1 OPERACIONAL Origem: SA5 + txt do Luciano quadros 6/7/8

O que e: cooperados.service.ts:836-901 aprovarConcessionaria cria notificacao interna e envia email - mas NAO dispara WhatsApp. Voce escreveu "avisa admin, avisa membro" - geralmente significa WhatsApp.

Impacto: Membro nao recebe a noticia mais aguardada do cadastro no canal que ele usa todo dia. Estimativa fix: 1-2h Code

5. Motor de realocacao inteligente entre multiplas usinas - 0% implementado

P1 ESTRATEGICO Origem: SA4 + txt do Luciano quadro 5

O que e: Voce descreveu: "se mais de uma usina folga insuficiente, soma folgas atendem - sistema reanalisa, troca cooperados entre usinas, gera listas, admin aprova lista por lista". Hoje motor escolhe primeira usina com vaga (linear). Migracao manual 1-a-1 funciona. Sem otimizacao global.

Impacto: Cooperado novo bloqueado em LISTA_ESPERA mesmo quando agregando folga de 3 usinas daria pra alocar. Admin faz manualmente. Estimativa: 16-24h Code (algoritmo + telas aprovacao por usina + historico)

6. Automacao concessionaria - 0% (quadros 6/7/8 do txt)

P1 OPERACIONAL Origem: SA4 + txt do Luciano

O que e: Voce descreveu: protocolo na concessionaria, polling site EDP, leitor email homologacao, fetcher MMGD da cooperativa, trigger "fatura GD recebida -> libera cobranca". Hoje admin digita protocolo manual via contratos.service.ts:399 ativar(). Email-monitor le faturas mas nao detecta aprovacao.

Impacto: Processo concessionaria 100% fora do sistema. Maior gargalo operacional. Estimativa: 2-3 sprints dedicados (escopo grande, vale fatiar)

7. Cadastro membro SEM UC invisivel na UI

P1 ESTRATEGICO Origem: SA1 + txt do Luciano quadro 1

O que e: Voce descreveu: "cooperado SEM UC que traga novos usuarios, destinar tokens ou converter em valor". Banco pronto (enum TipoCooperado.SEM_UC), service trata em 5 pontos, endpoint aceita. UI nunca expoe a escolha - wizard obriga upload de fatura, cadastro publico idem.

Impacto: Bloqueia caso de uso "rede de indicadores remunerada em tokens" (Sprint 8 + MST D-44). Estimativa: 4-6h Code (adicionar Step0 chooser + pular Steps quando SEM_UC)

8. Cadastro de Usina expandido - 0% no schema

P0 REGULATORIO Origem: SA4 + txt do Luciano adendo

O que e: Voce descreveu: classe GD I/II/III, dono (cooperado ou nao), forma aquisicao (cessao/aluguel/propria), forma pagamento dono (fixo/percentual), endereco completo, media mensal prevista distinta. Schema atual tem so basico (nome, potencia, capacidade, cidade/estado, proprietario texto).

Impacto: Caso Exfishes R$ 310k/ano ATIVO (D-30B P0) - sem campo classeGd, sem validacao em migracoes, risco financeiro real. Estimativa: Sprint 5 dedicado (4-6 dias Code)

9. Assinafy (assinatura digital) - zero codigo

P1 JURIDICO Origem: SA5

O que e: Mencionado 4x em PRODUTO.md como "Sprint 3 futura" + 1x em debitos-tecnicos.md. Hoje o aceite e "campos no model" - sem assinatura juridica de termo + procuracao. Zero integracao com provedor de assinatura.

Impacto: Aceite atual e fragil juridicamente. Membro pode contestar contrato. Estimativa: 2-3 sprints (integracao + UI + validacoes)

10. AuditLog (D-30N) inativo - 0 writes

P2 OBSERVABILIDADE Origem: SA6+SA7

O que e: Tabela audit_logs existe no schema (Sprint 13a Dia 1), campos completos (usuario, perfil, impersonating, acao, recurso, metadata, ip, userAgent). Zero prisma.auditLog.create no codigo. Sem interceptor, decorator, helper, module.

Impacto: Bloqueia Sugestao #7 (Observabilidade Total) que voce pediu pra ser lembrado. Sem rastreabilidade de quem fez o que. Estimativa: 6-8h Code (interceptor + decorator + helper + module + aplicar em endpoints sensiveis)

11. CoopereAI bot cita RN 482/2012 caducada

P1 REGULATORIO D-30I catalogado

O que e: Bot CoopereAI responde leads e cooperados desde 2026-04 com regulacao defasada (coopere-ai.service.ts:25 system prompt). RN 482 foi substituida pela Lei 14.300/2022.

Impacto: Toda interacao do bot transmite info regulatoria errada. Catalogado mas nao corrigido. Estimativa: 30min

12. FaturaSaas nao envia via Asaas (D-29F decomposto)

P1 RECEITA SISGD Origem: SA2+SA6

O que e: Cron mensal dia 1 6h cria FaturaSaas no banco. Mas: (1) Asaas nao recebe a cobranca; (2) parceiro nao recebe comunicacao D-7/D-3/D-1. ConfigGatewayPlataforma vazio em prod (0 registros). 3 FaturaSaas PENDENTES no banco invisiveis.

Impacto: SISGD nao cobra parceiros automaticamente. Receita propria do Luciano nao flui. Estimativa: 1-2 semanas Sprint 1

13. D-54 LancamentoCaixa PREVISTO faltante (Caminho A)

P1 CONTABILIDADE Catalogado 14/05

O que e: gerarCobrancaPosFatura nao cria LancamentoCaixa PREVISTO. So o Caminho B (cobranca manual) cria. 4 cobrancas piloto Sub-Fase A (DIEGO, CAROLINA, ALMIR, THEOMAX) ficaram sem lancamento. gerarCobrancasLote:1058 tem o mesmo gap.

Impacto: Relatorios financeiros sub-reportam receita prevista. Calculo da FaturaSaas SISGD usa receita prevista do parceiro - afeta cascata. Estimativa: 1-2h Code + saneamento retroativo
