Resumo executivo

Fluxo de cadastro de usina e alocacao de membros tem 6 fases: cadastro da usina (A), alocacao % kWh entre membros (B), listas aprovacao interna (C), envio concessionaria (D), aprovacao concessionaria (E), operacao recorrente (F). Estado real: cadastro basico funciona (~60%), mas 4 grandes blocos sao gap (0%) conforme descrito por Luciano no txt: cadastro expandido (classe GD, forma aquisicao, forma pagamento dono), realocacao inteligente multi-usina, automacao concessionaria, operacao recorrente (crons concentracao/queima/MMGD). Caso Exfishes R$ 310k/ano ativo (D-30B P0) por falta de campo classeGd.

Pontos criticos catalogados

D-30B P0 - Caso Exfishes R$ 310k/ano ativo

P0 REGULATORIO Origem: investigacao 20/04 Sprint 11

O que e: Schema Usina nao tem campo classeGd. Migracoes entre usinas nao validam se cooperado esta indo de GD I -> GD II (que tem regras tarifarias diferentes). Caso real Exfishes: cooperado migrado sem validacao gerou prejuizo retroativo estimado R$ 310k/ano em creditos mal compensados.

Fix: Sprint 5 schema regulatorio - adicionar enum ClasseGd + validacao em motor-proposta.aceitar + migracoes-usina. Custo: 4-6 dias Code dedicado.

Cadastro de Usina expandido = 0%

P0 Origem: adendo txt Luciano

O que falta: schema Usina nao tem classeGd, formaAquisicao (CESSAO/ALUGUEL/PROPRIA), formaPagamentoDono (VALOR_FIXO/PERCENTUAL_ARRECADACAO), valor/percentual do pagamento, endereco completo, mediaMensalPrevistaKwh distinta de producaoMensalKwh. UI nova (/dashboard/usinas/nova) so pede o basico.

Fix: migration schema + atualizar UI cadastro + atualizar UI detalhe. Custo: 6-8h Code.

Realocacao inteligente multi-usina = 0%

P1 Origem: quadro 5 do txt

O que falta: motor escolhe primeira usina com vaga (linear). Sem soma de folgas multi-usina, sem analise mesma distribuidora + classe GD, sem proposta-rascunho de remanejamento, sem aprovacao admin lista-por-lista, sem historico/relatorio.

Fix: Sprint dedicado 8. Algoritmo otimizacao + telas aprovacao por usina + historico MigracaoUsina ja existe (schema 1823). Custo: 16-24h Code.

Automacao concessionaria = 0%

P1 Origem: quadros 6/7/8 do txt

O que falta: sem geracao automatica de protocolo na EDP, sem polling site EDP por numero de protocolo, sem leitor de email de homologacao (regex), sem fetcher de relatorio MMGD da cooperativa, sem trigger "1a fatura GD chega -> libera cobranca automatico".

Fix: Sprint dedicado pos-canario AMAGES. Email-monitor ja existe, falta detector especifico. Custo: 2-3 sprints.

D-30C 1:1 Contrato↔Usina bloqueia UC com creditos multiplas usinas

P1

O que e: Schema atual nao suporta UC recebendo creditos de mais de 1 usina simultaneamente. Caso real possivel em GD II com multiplos arranjos de geracao distribuida.

Fix: schema novo ContratoUsina N:M + migration. Sprint 5.

D-30D sem dataProtocoloDistribuidora na UC

P1

O que e: Cooperado.protocoloConcessionaria existe (campo texto), mas data do protocolo nao registrada. Importante pra audit + calculo de prazo de aprovacao EDP.

Fix: migration + atualizar contratos.ativar. Sprint 5.

D-30F/G crons concentracao + queima saldo

P2

D-30F: sem cron de auditoria de concentracao > 25% por cooperado-usina (ANEEL exige limite).
D-30G: sem mecanismo de queima de saldo > 2 meses parado.

Fix: 2 crons novos. Custo: 4-6h cada.

Monitoramento Sungrow desativado

P2

O que e: monitoramento-usinas.service.ts:21 cron comentado desde Sprint 6 ("ativar Sprint 9+"). Telemetria de geracao em tempo real desativada.

Fix: reativar cron + validar credenciais Sungrow + tela monitoramento. Custo: 1-2 sprints.

D-30L Spec Fio B nunca implementada

P2

docs/specs/PROPOSTA-GD1-GD2-FIOB-2026-03-26.md tem tabela 2024-2029. Schema sem model RegrasFioB. Calculo Fio B 2026=60% hardcoded em codigo.

Fix: schema RegrasFioB + seed tabela ANEEL + service que le da tabela. Sprint 5.
