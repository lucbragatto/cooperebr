# Documentação CoopereBR

Estrutura dos documentos do projeto.

## Organização

- **`referencia/`** — descrições vivas do sistema: arquitetura,
  contexto, fórmulas de cobrança, sprints em andamento.
  Consultar aqui primeiro.

- **`specs/`** — propostas e especificações de features.
  Documentos escritos antes da implementação.

- **`changelog/`** — registros do que foi implementado, fases
  concluídas, bugfixes relevantes. Escritos depois da
  implementação.

- **`qa/`** — relatórios de QA diários e temáticos, auditorias,
  análises técnicas pontuais, histórico de qualidade.

- **`sessoes/`** — material de trabalho de sessões específicas
  (prompts, retomadas, tasks pontuais). Tende a ficar obsoleto
  rápido — limpar periodicamente.

## Para começar

Leia primeiro:
1. `referencia/CONTEXTO-CLAUDEAI.md` — visão geral do projeto
2. `referencia/ARQUITETURA-RESUMO.md` — mapa de entidades e fluxos
3. `referencia/SPRINT-BACKLOG-COMPLETO.md` — estado atual do roadmap

## Raiz do repositório

Ficam na raiz apenas:
- `README.md` — ponto de entrada do projeto
- `CLAUDE.md` — instruções de contexto pro assistente
