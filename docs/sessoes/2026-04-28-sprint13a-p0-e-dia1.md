## Incidentes do dia

### PM2 `cooperebr-backend` chegou a 331 restarts acumulados

Causas observadas em 2 momentos:
- **Manhã:** porta 3000 ocupada por node órfão (resíduo de sessões anteriores). PM2 spawnava novo processo, falhava com `EADDRINUSE`, restartava em loop. Resolvido com kill manual + `pm2 stop`/`start` limpo.
- **Noite:** Luciano reaproveitou comando velho do histórico do PowerShell ao reabrir VS Code, executou `pm2 stop/start` redundantes várias vezes — cada um incrementando o restart count.

Resultado: débito P3 registrado (`PM2 sem max_restarts`), regra operacional adicionada ao CLAUDE.md ("antes de rodar pm2 stop/start/restart, sempre `pm2 list` primeiro").

### Frontend Next.js dev caiu durante reinício do VS Code

Sintoma: browser mostrou "Cannot connect" em `localhost:3001` após reload de janela do VS Code. Causa: Next.js dev roda em terminal interativo (não gerenciado pelo PM2), e ao fechar o terminal o processo morreu junto.

Recuperação: `cd web ; npm run dev` em terminal novo. Regra adicionada ao CLAUDE.md ("Frontend Next.js dev — terminal vivo, NÃO gerenciado pelo PM2").

### Rotas antigas voltaram a funcionar após restart do Next.js dev

Após o reinício do frontend, algumas rotas antigas (`/dashboard/saas/planos`, `/dashboard/cooperativas`) pareciam dar erro. Era cache do Turbopack. Limpeza com restart limpo resolveu — não era bug, só cache stale.
