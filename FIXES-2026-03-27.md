# Fixes aplicados — 2026-03-27

## BUG 1 — Filtro de status ignorado no GET /cobrancas
- **Arquivos:** `cobrancas.controller.ts`, `cobrancas.service.ts`
- Adicionado query param `status` (string ou array, separado por vírgula)
- Service filtra com `where: { status: { in: [...] } }`

## BUG 2+3 — Cobranças PENDENTE vencidas sem multa/juros
- **Fix A** (`cobrancas.job.ts`): `marcarVencidas()` agora inclui `status: { in: ['A_VENCER', 'PENDENTE'] }` na transição para VENCIDO
- **Fix B** (`cobrancas.service.ts`): `darBaixa()` calcula multa/juros em tempo real também para PENDENTE com dataVencimento < hoje
- Lógica: `diasEfetivos = max(0, diasAtraso - carencia)`; `multa = valor * %multa`; `juros = valor * %jurosDia * diasEfetivos`

## BUG 5 — darBaixa sem metodoPagamento
- **Arquivos:** `cobrancas.controller.ts`, `cobrancas.service.ts`
- Campo opcional `metodoPagamento?: 'PIX' | 'BOLETO' | 'TRANSFERENCIA' | 'DINHEIRO'`
- Salvo no campo `observacoes` do LancamentoCaixa

## BUG 6 — Boleto sem linha digitável
- **Schema:** Adicionado campo `linhaDigitavel String?` em `AsaasCobranca`
- **Asaas service:** Após emitir BOLETO, busca `GET /payments/{id}/identificationField` e salva
- **WhatsApp:** Incluído nas mensagens de cobrança e abordagem de inadimplentes
- **Bot:** Adicionado `linhaDigitavel` no select de `buscarCobrancasPorTelefone`

## BUG 7 — Endpoint de reenvio individual
- **Novo endpoint:** `POST /cobrancas/:id/reenviar-notificacao`
- Verifica tenant, busca cooperado, calcula multa/juros se vencida
- Envia WhatsApp com PIX copia-e-cola, linha digitável e link
- Atualiza `whatsappEnviadoEm`
